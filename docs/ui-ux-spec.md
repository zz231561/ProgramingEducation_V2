# UI/UX 介面規格書

> 本文件定義 ProgramingEducation V2 的完整介面佈局、互動邏輯與視覺規格。
> Design Tokens（色彩/字型/圓角）詳見 `.claude/rules/frontend.md`，本文件不重複。

---

## 設計理念

採用 **VSCode 風格的 IDE 佈局**，但以教育場景為核心重新設計：
- 學生打開系統就像打開一個「學習用 IDE」，降低認知負擔
- 左側導覽提供功能切換，中央區域最大化工作空間，右側 Coddy隨時待命
- 所有頁面共享統一骨架，切換功能不離開主框架，保持沉浸感

**Avatar 展開選單：**

## 2. Workspace 頁面（程式碼工作區）

> 學生的主要使用頁面，佔 80% 以上使用時間

```
┌──────────┬───────────┬──────────────────────────────┬─────────────┐
│ Activity │ File Tree │  Toolbar                      │  AI Chat    │
│ Bar      │ (200px)   │  [main.cpp] [C++▾] [stdin][▶]│  Panel      │
│          │           ├──────────────────────────────┤             │
│          │ 📁 我的檔案│                              │  🤖 ...     │
│          │  📄 main  │  Code Editor (CodeMirror 6)  │             │
│          │  📄 hw1   │  ┌──────────────────────┐    │  👤 ...     │
│          │  📄 hw2   │  │ 1 │ #include <iostream>│   │             │
│          │           │  │ 2 │ using namespace std│   │  🤖 ...     │
│          │ 📁 課程範例│  │ 3 │                    │   │             │
│          │  📄 array │  │ 4 │ int main() {       │   │             │
│          │  📄 loop  │  │ 5 │   cout << "Hello"; │   │             │
│          │           │  │ 6 │   return 0;        │   │             │
│          │           │  │ 7 │ }                   │   │             │
│          │ [+ 新檔案]│  └──────────────────────┘    │             │
│          │           ├──────────────────────────────┤             │
│          │           │  Output Panel (可拖曳高度)    │  [輸入...]  │
│          │           │  [stdout|stderr|compile] tab  │  [Send ▶]  │
│          │           │  > Hello                      │             │
└──────────┴───────────┴──────────────────────────────┴─────────────┘
```

### 2.1 File Tree（檔案樹）

| 屬性 | 規格 |
|------|------|
| 寬度 | 200px 預設，可拖曳調整（最小 150px，最大 320px） |
| 背景 | `--bg-default` (#161B22) |
| 右邊框 | 1px `--border-muted` |
| 收合方式 | 點擊 Activity Bar 的 Workspace 圖示 toggle |

**檔案樹結構：**
- **📁 我的檔案**：學生自行建立的 `.cpp` 檔案
- **📁 課程範例**：從 Learn 頁面「在 Workspace 執行」帶入的範例程式碼（唯讀）
- **📁 作業**：教師指派的作業檔案

**操作：**
- `+ 新檔案` 按鈕：建立新 `.cpp` 檔（預設命名 `untitled_1.cpp`）
- 右鍵選單：重新命名、刪除、複製
- 拖曳排序：可在資料夾間拖曳移動
- 當前開啟的檔案：文字色 `--text-primary`，背景 `--bg-subtle`
- 未儲存修改：檔名旁顯示白色圓點

### 2.2 Toolbar（工具列）

| 屬性 | 規格 |
|------|------|
| 高度 | 40px |
| 背景 | `--bg-default` (#161B22) |
| 下邊框 | 1px `--border-default` |

**Toolbar 元素（左至右）：**

| 元素 | 說明 |
|------|------|
| 檔案名稱 | 顯示當前開啟的檔案名稱（如 `main.cpp`） |
| 語言選擇 | `C++` 下拉選單（Phase 1 僅 C++，預留擴充） |
| spacer | 彈性空間 |
| stdin 按鈕 | 點擊展開 stdin 輸入面板 |
| ▶ Run 按鈕 | `--btn-primary-bg` 綠色，執行程式碼 |

### 2.3 Code Editor 區域

| 屬性 | 規格 |
|------|------|
| 編輯器 | CodeMirror 6 |
| 主題 | One Dark（與 VSCode Dark+ 接近） |
| 背景 | `--bg-canvas` (#0D1117) |
| 字型 | JetBrains Mono, 14px, line-height 1.6 |
| 行號 | 顯示，色彩 `--text-muted` |
| 功能 | 語法高亮、自動補全、括號匹配、縮排引導線 |
| 佔比 | 預設佔 Content Area 垂直空間的 70% |

### 2.4 Output Panel（輸出面板）

| 屬性 | 規格 |
|------|------|
| 背景 | `--bg-inset` (#010409) |
| 上邊框 | 1px `--border-default`，可拖曳調整高度 |
| 預設高度 | Content Area 垂直空間的 30% |
| 最小高度 | 收合為一行 status bar (28px) |
| Tab | `stdout` / `stderr` / `compile`，active 有底線 |

**Tab 行為：**
- `stdout`：程式標準輸出，白色文字
- `stderr`：標準錯誤，`--accent-red` 文字，有新內容時 tab 顯示紅點
- `compile`：編譯器訊息，可點擊跳轉至對應行號

**Status bar（收合狀態）：**
- 顯示最近一次執行結果：`✓ Passed (0.12s)` 或 `✗ Error` 或 `⏳ Running...`
- 點擊展開完整 Output Panel

### 2.5 stdin 面板

- 點擊 Toolbar `stdin` 按鈕後，在 Editor 上方滑出輸入區
- 多行文字輸入，高度 80px，可拖曳調整
- placeholder: `輸入測試資料（每行一筆）...`
- 執行時自動注入 Judge0 stdin

### 2.6 Run 流程動效

1. 點擊 `▶ Run` → 按鈕變為 `⏳ Running...`，禁用重複點擊
2. Output Panel 自動展開（若收合）
3. 編譯成功 → stdout tab 顯示結果
4. 編譯失敗 → 自動切到 compile tab，錯誤行在 Editor 標紅底線
5. 執行結果自動注入 AI Chat context（背景，不顯示在對話中）

### 4.1 測驗流程

1. **開始前**：顯示測驗資訊（題數、預估時間、涵蓋概念）
2. **作答中**：單題顯示，底部進度條，右上計時器
3. **提示機制**：點擊 `💡 提示` → 展開右側 AI Chat，AI 根據 Hint Ladder 給予引導
4. **程式撰寫題**：中央區域切換為內嵌 CodeMirror 編輯器 + 執行按鈕
5. **結果頁**：完成後顯示分數、各題正誤、概念精熟度變化
